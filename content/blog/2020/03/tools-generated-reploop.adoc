从Github Pages开始，静态资源建站开始流行起来了。如果你仅仅是想写点东西，那这就完全够用了。所以现在弄个小站，根本不需要服务器(如Linux)，数据库(如MySQL)，应用程序(比如Tomcat，Apache，Nginx等)，编程语言(如JSP，PHP，SpringMVC等)等等统统都可以不需要学，也不需要会。

LAMP vs SSM

云时代对小不点的企业或者个人来说啥都可以找到云解决方案，比如给网站加个评论那就用 https://disqus.com/[DISQUS]，想有个统计那就用 https://analytics.google.com/analytics/web/[Google Analytics]，想上个CDN加速那就用 https://www.cloudflare.com/[Cloudflare]，域名也可以不用操心，就用Github Pages的，有自己域名的也可以用。HTTPS证书也有免费的 https://letsencrypt.org/[Let's Encrypt]。只要搭好架子，剩下的就是使用熟悉的格式如 https://www.markdownguide.org/[Markdown]或者 https://asciidoctor.org/[Asciidoctor]等开始写就可以了。

== Asciidoctor

英文句子中的每个单词前后都有空格，但是中文句子却不是。然后一些加粗必须和前面的字有一个空格Asciidoc才能解析。 比如 `This is *bold* .`很正常，
但是 `这是*加粗*文字` 这个不能正确解析，需要加入空格才可以，像这样：`这是 *加粗* 文字`。不过这样的话出来的文字就有空格了。

=== 本地测试

执行以下命令，并启动本地Jetty服务器，方便在浏览器中预览。

[source,bash]
----
jbake -b -s
----

但是发布的时候不要 `-s` 选项， 否则生成的feed.xml文件中的链接地址就会变成 `http://localhost:8820/`。
